# AI Email Scheduling Assistant (Hackathon MVP)

Minimal end-to-end prototype:
1. Read participant availability from Gmail messages (Gmail API)
2. Parse simple availability text into time windows (IST)
3. Store windows in SQLite
4. Find overlapping windows across 2-3 participants
5. Create a Google Calendar event
6. Send confirmation replies via Gmail (with required AI disclaimer)

This MVP intentionally keeps parsing simple and assumes all times are in `Asia/Kolkata` (IST).

## Project layout

- `app/main.py`: FastAPI endpoints
- `app/email/gmail_client.py`: Gmail read/send helpers
- `app/parsing/availability_parser.py`: Parses availability from email text
- `app/scheduling/overlap_finder.py`: Finds overlapping time windows
- `app/calendar/google_calendar_client.py`: Creates calendar events
- `app/db/*`: SQLite initialization + CRUD
- `app/google/oauth.py`: OAuth credential loading
- `scripts/auth_google.py`: One-time OAuth token setup

## Setup (step-by-step)

1. Open a PowerShell terminal in this folder: `C:\Users\SWAMI\ai-email-scheduling-assistant`
2. Create a virtualenv and activate it:
   - `python -m venv .venv`
   - `.venv\\Scripts\\Activate.ps1`
3. Install dependencies:
   - `pip install -r requirements.txt`
4. Create a `.env` file from the template:
   - Copy `.env.example` to `.env` and adjust paths if needed
5. Add your Google OAuth "Client ID" credentials:
   - In Google Cloud Console, enable the `Gmail API` and `Google Calendar API`
   - Create an OAuth Client ID (use an OAuth client that allows `http://localhost` redirects)
   - Download the OAuth client secrets JSON and save it as `credentials.json` in the project root
     (or update `GOOGLE_CLIENT_SECRETS_PATH`)
6. Run OAuth token setup once:
   - `python scripts/auth_google.py`
   - This creates `data/token.json`
7. Start the API:
   - `uvicorn app.main:app --reload --port 8000`

## How to run the prototype

### 1) Get Gmail `message_ids`

You can obtain message IDs from the Gmail web UI or by manually listing messages via Gmail API.

### 2) Call the endpoint

POST `http://localhost:8000/process-emails`

Request JSON:
```json
{
  "message_ids": ["185c...","185d..."],
  "organizer_email": "your-team-email@example.com"
}
```

Response includes:
- parsed overlap start/end
- created Calendar event id + link

### 3) What the parser supports (MVP)

Examples it can parse:
- `I am free tomorrow 10-12`
- `Available at 3 PM`
- `Free today 10-12 PM` (if AM/PM is included)

Rules:
- If the text doesn’t mention `today` or `tomorrow`, it defaults to `tomorrow`.
- If only one time is given (e.g. `3 PM`), it assumes a 1-hour slot (`3:00-4:00`).
- If a range is given (e.g. `10-12` or `10 AM-12 PM`), that range is used.

Timezone assumption:
- For MVP we treat IST as a fixed `UTC+05:30` offset across all dates.

## Disclaimer requirement

Every confirmation email includes:
> This email was generated by an experimental AI assistant.

## Notes / limitations

- Hard-coded timezone assumption: `Asia/Kolkata`.
- Parsing is regex-based and intentionally limited for hackathon speed.
- Gmail parsing extracts `text/plain` first; it falls back to a best-effort `text/html` -> text conversion.
